# Lumi Books

Сайт для чтения книг в спокойном Apple-like стиле: стеклянный интерфейс, обложки, подборки, полные демо-книги, учетные записи и админская панель для добавления, редактирования и удаления книг.

## Запуск локально

```bash
node src/server.js
```

Откройте `http://localhost:3002`.

Демо-админ создается автоматически, если переменные окружения не заданы:

```text
email: admin@lumi-books.local
password: ChangeMe123!
```

Для продакшена обязательно задайте свои значения:

```bash
ADMIN_EMAIL=you@example.com
ADMIN_PASSWORD=strong-password
NODE_ENV=production
DATA_DIR=/var/data
node src/server.js
```

## Возможности

- регистрация и вход в учетную запись;
- сессии через httpOnly cookie;
- роль `reader` для новых пользователей;
- роль `admin` для добавления, редактирования и удаления книг;
- серверная проверка прав на всех admin API;
- чтение книг только после входа;
- локальные SVG-обложки для демо-книг;
- подборки книг через `/api/collections`;
- главы и полные версии оригинальных демо-текстов;
- импорт обычных `.txt` книг из админки;
- автоматическое распознавание глав вида `Глава I`, `Глава 1`, `Пролог`, `Эпилог`;
- постраничное чтение с кнопками, клавишами стрелок и переходом по главам;
- поиск и фильтр по категориям;
- настройка размера текста в режиме чтения;
- файловое хранилище в `DATA_DIR/library.json`.

## Деплой в облако

Проект готов к деплою на Render через `render.yaml`.

1. Создайте новый Web Service из Git-репозитория.
2. Render подхватит `render.yaml`.
3. Укажите секреты `ADMIN_EMAIL` и `ADMIN_PASSWORD`.
4. Persistent disk уже описан в конфиге и монтируется в `/var/data`.
5. После деплоя первый админ будет создан при старте приложения.

Также можно запускать через Docker:

```bash
docker build -t lumi-books .
docker run -p 3002:3002 -e PORT=3002 -e ADMIN_EMAIL=you@example.com -e ADMIN_PASSWORD=strong-password -v lumi-books-data:/data lumi-books
```

## Структура

- `src/server.js` - Node.js сервер, API, авторизация, роли, хранение данных.
- `src/seed.js` - полные демо-книги и подборки.
- `public/covers/` - локальные SVG-обложки книг.
- `public/index.html` - основная HTML-страница.
- `public/styles.css` - стеклянный Apple-like интерфейс и адаптивная верстка.
- `public/app.js` - клиентская логика библиотеки, чтения, auth и админки.
- `render.yaml` - облачный деплой на Render.
- `Dockerfile` - контейнерный запуск.
